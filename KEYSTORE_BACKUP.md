# 🔑 Android 签名密钥备份指南

## ⚠️ 极其重要！

**签名密钥是你应用的"身份证"**
一旦丢失，你将**永远无法更新**已发布的应用！

---

## 📋 需要备份的文件和信息

### 1. 密钥文件
```
android-release.keystore
```
- 位置：项目根目录
- 大小：约 2-3 KB
- 类型：二进制文件

### 2. 密钥信息（文本）
```
密钥库文件：android-release.keystore
密钥库密码（storePassword）：[你设置的密码]
密钥别名（keyAlias）：voicica-key
密钥密码（keyPassword）：[你设置的密码]
创建日期：[密钥生成日期]
有效期：10000 天（约 27 年）
```

---

## 🔒 推荐备份方案

### 方案 A: 密码管理器（推荐）⭐

**适用工具**: 1Password, Bitwarden, LastPass

**步骤**：
1. 创建新条目："Voicica Android Release Key"
2. 上传密钥文件作为附件
3. 在笔记中保存所有密钥信息
4. 添加标签："android", "签名密钥", "重要"

**优点**：
- ✅ 加密存储
- ✅ 多设备同步
- ✅ 可设置紧急联系人
- ✅ 自动备份

### 方案 B: 云盘 + 加密（推荐）⭐

**适用工具**: Google Drive, iCloud, OneDrive

**步骤**：
1. 使用 7-Zip / WinRAR 创建加密压缩包
   ```bash
   # 使用强密码加密
   7z a -p -mhe=on -mx=9 voicica-keystore-backup.7z android-release.keystore
   ```
2. 上传到云盘的安全文件夹
3. 密码保存到密码管理器

**优点**：
- ✅ 双重加密（压缩包密码 + 云盘账户）
- ✅ 跨平台访问
- ✅ 自动版本历史

### 方案 C: 多地物理备份

**适用场景**: 极端安全需求

**步骤**：
1. 复制密钥文件到 2-3 个 U 盘
2. 在不同物理位置存放（家里、办公室、银行保险箱）
3. 密码信息单独保存（纸质 + 密码管理器）

## 🆘 恢复流程

### 如果需要从备份恢复

1. **从备份中取出密钥文件**
   ```bash
   # 解压加密压缩包（如果使用了加密）
   7z x voicica-keystore-backup.7z
   ```

2. **放置到项目根目录**
   ```bash
   cp android-release.keystore /path/to/project/
   ```

3. **配置环境变量**
   编辑 `.env.local`：
   ```env
   ANDROID_KEYSTORE_PASSWORD=your_password
   ANDROID_KEY_PASSWORD=your_password
   ```

4. **验证密钥可用**
   ```bash
   keytool -list -v -keystore android-release.keystore
   # 输入密码后应显示密钥信息
   ```

5. **测试构建**
   ```bash
   npm run android:build
   ```

## 💡 最佳实践

### 定期检查

**建议频率**: 每年一次

**检查内容**：
- [ ] 备份文件完整性
- [ ] 密码仍然可用
- [ ] 云盘账户未过期
- [ ] U 盘硬件完好

### 团队协作

如果是团队开发：
- 指定 2-3 人掌握密钥（负责人 + 备份负责人）
- 使用企业级密码管理器（1Password Teams）
- 明确密钥使用权限和流程

### 密钥轮换

**注意**: 一旦发布应用，**不能更换密钥**！

唯一例外：
- 应用尚未发布
- 愿意更换包名（相当于新应用）
- 通过 Google Play App Signing（推荐）

## 📞 紧急联系方式

### 如果密钥丢失

**Google Play 官方政策**：
- ❌ 无法恢复丢失的密钥
- ❌ 无法更新已发布的应用
- ✅ 可以使用新密钥发布"新应用"（不同包名）

**建议行动**：
1. 立即检查所有备份位置
2. 联系团队成员确认是否有备份
3. 如确认丢失：
   - 考虑使用新包名重新发布
   - 向用户说明需要卸载旧版重新安装
   - 更新所有营销材料

## 🔐 使用 Google Play App Signing（推荐）

### 什么是 Play App Signing？

Google 帮你管理签名密钥：
- Google 保管"App Signing Key"（真正的发布密钥）
- 你只需保管"Upload Key"（上传密钥）
- 如果 Upload Key 丢失，可以向 Google 申请重置

### 如何启用？

1. 首次上传应用到 Google Play Console
2. 选择"Let Google manage and protect your app signing key"
3. Google 会生成 App Signing Key
4. 你的密钥变为 Upload Key（可重置）

**优点**：
- ✅ 密钥丢失可恢复
- ✅ Google 管理更安全
- ✅ 支持 App Bundles 优化
