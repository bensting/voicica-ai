# App 自动更新机制

本文档说明 Android App 的版本检查和自动更新功能。

## 版本号说明

App 版本由两部分组成：

| 字段 | 说明 | 示例 |
|------|------|------|
| `version` | 版本号，用于显示给用户 | `"1.0.0"` |
| `buildNumber` / `version_code` | 构建号，用于版本比较 | `29` |

**重要：版本更新判断只看 `version_code`，与版本号无关。**

```json
// native-version.json
{
  "version": "1.0.0",
  "buildNumber": 29
}
```

### 版本比较示例

| 版本号 | version_code | 是否提示更新 (当前 version_code=29) |
|--------|-------------|----------------------------------|
| 1.0.0  | 29          | ❌ 不更新（相同） |
| 1.0.1  | 30          | ✅ 需要更新 (30 > 29) |
| 2.0.0  | 28          | ❌ 不更新 (28 < 29) |

## 更新流程

```
┌─────────────────────────────────────────────────────────────┐
│                      App 启动                                │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼ (延迟 2 秒)
┌─────────────────────────────────────────────────────────────┐
│  1. 检测是否原生 App (Capacitor.isNativePlatform())         │
│     - Web 浏览器：跳过更新检查                               │
│     - 原生 App：继续                                        │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  2. 获取当前 App 版本信息                                    │
│     App.getInfo() → { version: "1.0.0", build: "29" }       │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  3. 调用服务器 API 检查更新                                   │
│     checkAppUpdate('android', 29)                           │
│     → 查询 app_releases 表，找 is_latest=true 的记录         │
│     → 比较 version_code > 29 ?                              │
└─────────────────────────────────────────────────────────────┘
                              │
                    ┌─────────┴─────────┐
                    │                   │
                有新版本            无新版本
                    │                   │
                    ▼                   ▼
┌──────────────────────────┐   ┌──────────────────┐
│  4. 显示更新弹窗          │   │    静默结束      │
│     AppUpdatePrompt      │   └──────────────────┘
│     - 显示版本号          │
│     - 显示更新内容        │
│     - 强制/可选更新       │
└──────────────────────────┘
                    │
                    │ 用户点击"立即更新"
                    ▼
┌─────────────────────────────────────────────────────────────┐
│  5. 检查安装权限                                             │
│     AppInstallPlugin.canInstallUnknownApps()                │
│     - 未授权：跳转系统设置页面                                │
│     - 已授权：继续下载                                       │
└─────────────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────────┐
│  6. 下载 APK 文件                                            │
│     FileTransfer.downloadFile()                             │
│     - 下载到 Cache 目录                                      │
│     - 显示下载进度条                                         │
└─────────────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────────┐
│  7. 触发安装                                                 │
│     AppInstallPlugin.installApk()                           │
│     - 打开系统安装界面                                       │
└─────────────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────────┐
│  8. 用户确认安装                                             │
│     - App 关闭                                               │
│     - 新版本安装完成                                         │
│     - 用户重新打开 App                                       │
└─────────────────────────────────────────────────────────────┘
```

## 数据库表结构

`app_releases` 表：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | Int | 主键 |
| `platform` | String | 平台：`android` / `ios` |
| `version` | String | 版本号，如 `"1.0.0"` |
| `version_code` | Int | **构建号，用于版本比较** |
| `download_url` | String | APK 下载链接 |
| `file_size` | BigInt | 文件大小（字节） |
| `release_notes` | String? | 更新说明 |
| `is_latest` | Boolean | **是否为最新版本（同平台只有一个为 true）** |
| `is_force_update` | Boolean | 是否强制更新 |
| `is_active` | Boolean | 是否启用 |
| `download_count` | Int | 下载次数 |
| `created_at` | DateTime | 创建时间 |

### 关键字段说明

- **`version_code`**: 必须递增，每次发布新版本都要 +1
- **`is_latest`**: 同一平台只能有一条记录为 `true`，用于标识最新版本
- **`is_force_update`**: 为 `true` 时用户无法跳过更新
- **`is_active`**: 可用于下架某个版本

## 发布新版本流程

### 1. 更新版本号

编辑 `native-version.json`：

```json
{
  "version": "1.0.1",
  "buildNumber": 30  // 必须比上一版本大
}
```

### 2. 同步到 Android 项目

```bash
npm run native:version:sync
```

### 3. 构建 APK

```bash
npm run android:build
```

### 4. 上传到后台

在管理后台上传 APK 文件，填写：
- 版本号：`1.0.1`
- 构建号：`30`
- 更新说明（可选）
- 是否强制更新
- 设为最新版本 ✅

### 5. 验证

旧版本 App 打开后会自动检测到新版本并弹出更新提示。

## 测试方法

### 使用测试环境

构建连接测试服务器的 APK：

```bash
npm run android:build:test test
```

这会构建一个连接到 `https://www.ai-voice-labs.com/studio` 的测试包。

### 测试流程

1. 在测试环境数据库的 `app_releases` 表中添加一条记录：
   - `platform`: `android`
   - `version`: `1.0.1`
   - `version_code`: `30` (比当前 APK 的 buildNumber 大)
   - `is_latest`: `true`
   - `is_active`: `true`
   - `download_url`: 有效的 APK 下载链接

2. 安装测试 APK 到手机（当前 buildNumber 为 29）

3. 打开 App，2 秒后应该弹出更新提示

### 预设测试环境

| 环境名 | URL |
|--------|-----|
| `test` | `https://www.ai-voice-labs.com/studio` |
| `staging` | `https://staging.voicica.ai/studio` |
| `production` | `https://www.voicica.ai/studio` |

## 强制更新 vs 可选更新

| 类型 | `is_force_update` | 用户行为 |
|------|------------------|---------|
| 可选更新 | `false` | 可点击"稍后"关闭弹窗，下次启动仍会提示 |
| 强制更新 | `true` | 无法关闭弹窗，必须更新才能使用 App |

## 注意事项

1. **version_code 必须递增**：每次发布新版本，buildNumber 必须比之前的大
2. **只能有一个最新版本**：同一平台的 `is_latest` 只能有一条为 `true`
3. **签名一致**：测试包和生产包使用相同的 keystore 签名，可以覆盖安装
4. **网络依赖**：更新检查需要网络连接，无网络时静默跳过
5. **延迟检查**：App 启动后延迟 2 秒检查更新，避免影响启动体验
